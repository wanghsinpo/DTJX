#Options and defaults.ini file priority
You can designate options as command argument or as
defaults in defaults.ini file.

The options designated as command argument will write
over the defaults in defaults.ini file. This means
options has higher priority.

#log_analyzer Option List
-inputfile   : input log file
-tel         : TEL prober
-tsk         : TSK prober
-port        : GEMPORT log file
-prober      : GEMPROBER log file
-noreports   : do not show CEID reports
-norawdata   : do not include the original SECS messages
-options     : this message

#Usage
Example for TEL, gemport.log file:

log_analyzer -tel -port > /tmp/outputfile.txt

Since an input file is not provided,
the tool will display a file dialog to allow
you to choose a log file.

if you provided an input file, no dialog will be 
displayed.

log_analyzer -tel -port -inputfile gemport.log > /tmp/outputfile.txt

#defaults.ini
The all of option can be set in "defults.ini" file.
***********************************************************
# SPECS-FA Log Analyzer configuration file
 
# Prober Type - Specify ; tel(;equivalent -tel) or
#                         tsk(;equivalent -tsk) 
VGEM proberType         tel

# Object Log File Type; port(;equivalent -port) or
#                       prober(;equivalent -prober)
VGEM logType            port

# Default Directory Path for Log File Browser
VGEM logfileDir         /usr/tmp

# Show CEID reports; NO(;equivalent -noreports) or YES 
VGEM showReports        YES

# Show raw data; NO(;equivalent -norawdata) or YES
VGEM includeRawdata     YES
******************************************************

The tool will display a file dialog to allow
you to choose a log file. You can designate default directory 
path, where the file dialog will open first, into "logfileDir" flag.
The default is "/usr/tmp".

The "showReports NO" setting is equivalent to option "-noreports" and 
allows you to skip the reports attached to the CEIDs, therefore, 
reducing the size of the output file.

The "includeRawdata NO" setting is equivalent to option "-norawdata".
and the original SECS messages will be included along with the 
translated version. 

#Tips
To make output file small, we recommend you to extract necessary 
portion to analyze from the original gemprober.log or gemport.log
in advance.
